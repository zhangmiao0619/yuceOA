# OA系统 — 项目管理模块测试用例

| 文档版本 | 修改日期 | 修改人 | 修改说明 |
|---------|---------|-------|---------|
| v1.0 | 2026-05-21 | 测试组 | 初版，覆盖项目CRUD/子任务生命周期/暂停归档/通知/文件/权限 |

---

## 测试目标

- 验证项目管理模块的功能完整性与正确性
- 覆盖正常流程、异常流程、边界条件、权限隔离
- 确保子任务状态机（assigned→received→submitted→completed/rejected）各节点流转符合业务规则
- 确认数据权限（行级隔离）生效，角色间不可越权操作
- 验证通知推送在关键节点正确触达
- 验证文件上传功能的可用性和容量限制

## 测试范围

### 纳入范围

| 模块 | 子模块 |
|------|--------|
| 项目基础CRUD | 创建、编辑、查看（列表+详情）、删除 |
| 子任务生命周期 | 分配、接收、提交成果、审批（通过/驳回） |
| 项目高级操作 | 暂停申请与审核、归档 |
| 通知系统 | 通知创建、未读计数、已读标记、删除 |
| 文件上传 | 单文件、批量上传、文件大小校验 |
| 数据权限 | 行级数据隔离、操作权限校验 |
| 异常场景 | 网络超时、重复提交、Token过期、字段校验 |

### 排除范围

- 企业微信消息推送（依赖外部服务）
- 浏览器兼容性（仅覆盖 Chrome + Safari 核心功能）
- 非功能测试（性能、安全渗透、高并发）

## 风险分析

| 风险项 | 影响 | 概率 | 缓解措施 |
|--------|------|------|---------|
| SQLite 并发写入冲突 | 多人同时操作时数据不一致 | 低 | 后端使用事务（BEGIN/COMMIT） |
| 前端上传大文件导致浏览器卡死 | 测试效率下降 | 低 | 3G 限制 + 前端使用分片上传 |
| Token 过期后用户未感知 | 操作失败无反馈 | 中 | 实现 401 全局拦截提示 |
| 企业微信通知可达性 | 用户收不到通知 | 高（依赖外部） | 数据库通知兜底，前端角标显示 |
| 数据库迁移字段缺失 | 接口报 500 | 中 | 上线前执行 migrate 脚本验证 |
